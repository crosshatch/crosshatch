# X402 Primer

X402 specifies a standard message format for cryptocurrency invoicing and payment. When a merchant (for example, a web
server) requires payment for a resource, it responds with a **payment requirement**. Buyers use this machine-readable
pricing information to create payment payloads, which are effectively signed "checks." The merchant can verify and
deposit these payloads and serve the paid resource.

Crosshatch makes it easy for app developers to create payment payloads from secure user wallets; Crosshatch solves
cryptocurrency onboarding and abstracts over the task of intercepting and paying 402s.
